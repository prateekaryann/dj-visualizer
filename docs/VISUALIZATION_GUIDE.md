## 1. Julia Set Fractals (3D)

### What is a Julia Set?
A Julia set is a fractal generated by iterating a complex polynomial function $f(z) = z^2 + c$ for each pixel/voxel in space.

**Mathematical Definition:**
$$J(f_c) = \{z \in \mathbb{C} : \text{iterates of } f_c(z) = z^2 + c \text{ remain bounded}\}$$

### Implementation Strategy
- **Rendering Method**: GPU-accelerated raymarching via fragment shaders
- **Distance Field**: Use Distance Estimation Method (DEM) for efficient visualization
- **Smooth Color**: Apply iteration smoothing formula: $\mu = k + 1 - \frac{\log(\log|z_k|)}{\log(n)}$
- **Audio Reactivity**: Modulate the complex constant `c` parameter with frequency bins

### Key Advantages
✨ Mathematically beautiful self-similar structures  
✨ Infinite detail at any zoom level  
✨ Creates organic, flowing patterns when parameters animate  
✨ Low geometry overhead (just a quad with shader)  

### Three.js Implementation
```glsl
// Fragment shader pseudocode
void main() {
    vec2 c = uFrequency.xy;  // Modulated by audio
    vec3 pos = rayDir * t;   // Ray through voxel
    
    for (int i = 0; i < MAX_ITER; i++) {
        if (length(pos) > 2.0) break;
        pos = vec3(pos.x*pos.x - pos.y*pos.y, 2.0*pos.x*pos.y, pos.z*pos.z) + c.xyy;
    }
    
    float iteration = float(i);
    gl_FragColor = colorByIteration(iteration);
}
```

## 2. Parametric Surfaces (Audio-Reactive)

### What are Parametric Surfaces?
Surfaces defined by mathematical functions in two parameters: $\mathbf{r}(u,v) = (x(u,v), y(u,v), z(u,v))$

### Common Examples
| Surface | Formula | Properties |
|---------|---------|-----------|
| **Torus** | $x = r\sin v$, $y = (R+r\cos v)\sin u$, $z = (R+r\cos v)\cos u$ | Ring-like, smooth |
| **Sphere** | $x = \cos\theta\sin\phi$, $y = \sin\theta\sin\phi$, $z = \cos\phi$ | Perfect symmetry |
| **Surface of Revolution** | $\mathbf{r}(u,\phi) = (u\cos\phi, u\sin\phi, f(u))$ | From 2D curve |
| **Möbius Strip** | One-sided surface with twist | Non-orientable |
| **Klein Bottle** | 4D object projected to 3D | Complex topology |

### Audio Mapping Strategy
1. **Frequency Analysis**: Divide audio spectrum into bands (bass, mids, highs, ultra)
2. **Parameter Displacement**: Scale vertices along surface normals by frequency magnitude
3. **Wave Deformation**: Animate parameters `u,v` based on beat timing
4. **Color Cycling**: Assign colors based on frequency band energy

### Curvature-Based Coloring
- **Positive Curvature** (K > 0): Elliptic regions → Warm colors (orange, red)
- **Negative Curvature** (K < 0): Hyperbolic regions → Cool colors (cyan, blue)
- **Zero Curvature** (K = 0): Parabolic regions → Neutral colors

## 3. Organic Blobs / Metaballs

### Mathematical Foundation
Implicit surface defined by: $\sum_i f_i(d_i) \geq \text{threshold}$

Where $d_i$ is the distance from point to metaball center $i$.

### Contribution Functions

#### Simple Inverse Distance
$$f(r) = \frac{a}{r}$$
- Fastest, but has singularity at r=0
- Creates sharp peaks

#### Gaussian Potential
$$f(r) = a \cdot e^{-br^2}$$
- Smooth falloff
- Naturally bounded

#### Wyvill Brothers' Polynomial (Professional Standard)
$$f(r) = \begin{cases} 
a(1 - \frac{r^2}{R^2})^3 & \text{if } r < R \\
0 & \text{otherwise}
\end{cases}$$
- Smooth C² continuity
- Zero derivative at boundary
- Used in film industry (Pixar, DreamWorks)

### Audio Reactivity
- **Bass**: Modulate metaball radius
- **Mids**: Control number of active blobs
- **Highs**: Fine-grained deformation and jitter
- **Beat**: Spawn new blobs or explode existing ones

### Rendering Options
| Method | GPU Cost | Quality | Notes |
|--------|----------|---------|-------|
| **Marching Cubes** | Medium | High | Generates mesh per frame |
| **Raycasting** | High | Very High | Per-pixel raycasting |
| **Screen-Space SDF** | Low | Medium | Fast approximation |

## 5. Orbital Systems

### Concept
Particles orbiting a central attractor, with orbits deformed by audio frequencies.

### Physics
- **Gravitational Force**: $F = -\frac{GMm}{r^2}$
- **Audio Modulation**: Vary G (gravitational constant) with bass
- **Precession**: Orbit rotation rate synced to BPM

### Visualization Features
- Particle trails showing orbital paths
- Color gradient from center (cooler) to edges (warmer)
- Collision-based burst effects on beat

## Audio Analysis Techniques

### FFT (Fast Fourier Transform)
```javascript
const analyser = audioContext.createAnalyser();
analyser.fftSize = 2048;  // Power of 2: 512, 1024, 2048, 4096
const frequencyData = new Uint8Array(analyser.frequencyBinCount);
analyser.getByteFrequencyData(frequencyData);
```

**Frequency Mapping**: Bin index $i$ corresponds to frequency:
$$f = i \times \frac{\text{sampleRate}}{2048}$$

For 48kHz sample rate: 1 bin ≈ 23.4 Hz

### Onset Detection (Beat Detection)
```javascript
// Simple peak detection: compare current to smoothed history
const smoothingFactor = 0.85;
const threshold = 1.3;

for (let i = 0; i < frequencyData.length; i++) {
  const current = frequencyData[i];
  const smoothed = smoothingFactor * smoothHistory[i];
  
  if (current > smoothed * threshold) {
    // Beat detected in bin i
    onBeat(i);
  }
  smoothHistory[i] = (current + smoothed) / 2;
}
```

### Energy Distribution
```javascript
// Separate audio into bands
const bassEnergy = sum(frequencyData.slice(0, 5));        // 0-115 Hz
const midsEnergy = sum(frequencyData.slice(5, 20));      // 115-470 Hz
const highsEnergy = sum(frequencyData.slice(20, 40));    // 470-940 Hz
const ultraEnergy = sum(frequencyData.slice(40, 64));    // 940Hz+
```

## Color Schemes

### Adaptive Color Mapping
Assign color based on dominant frequency:
- **Red/Orange** (0-100 Hz): Bass-driven warmth
- **Yellow/Lime** (100-500 Hz): Mid-range energy
- **Cyan/Blue** (500-2kHz): High-frequency sparkle
- **Magenta/Purple** (2kHz+): Ultra-high presence

### Beat-Based Color Cycling
Cycle through a color palette on beat:
```javascript
const beatColors = [
  { r: 1.0, g: 0.1, b: 0.0 },  // Hot red
  { r: 1.0, g: 0.5, b: 0.0 },  // Orange
  { r: 0.0, g: 0.8, b: 1.0 },  // Cyan
  { r: 1.0, g: 0.0, b: 1.0 },  // Magenta
];
```
